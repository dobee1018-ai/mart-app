# 특가 제보/관리자 승인 데이터 흐름

## 목표

앱의 제보 기능은 "가격 정보를 긁어 모으는 구조"가 아니라, 동네 사용자가 좋은 특가를 공유하고 관리자가 검증한 정보만 앱에 노출하는 구조로 운영한다.

## 기본 흐름

1. 사용자가 앱에서 전단지, 문자, 영수증, 매장 가격표, 상품 사진 기반으로 특가를 제보한다.
2. 제보 원본은 `reports` 컬렉션에 `pending` 상태로 저장된다.
3. 관리자는 관리자 화면에서 제보 사진/내용/마트/가격/행사 유형을 확인한다.
4. 승인된 제보만 앱 노출 후보가 된다.
5. 전단지 제보는 `flyers`로, 전단지 안의 개별 상품은 `dealItems`로 분리 등록한다.
6. 문자/개별 상품 사진 제보는 승인 후 `dealItems`에 연결한다.

## 컬렉션 역할

### reports

사용자가 올린 원본 제보를 저장한다. 앱에 바로 노출하지 않는다.

주요 필드:

- `reporterId`: 제보자 uid
- `reporterNickname`: 앱에 표시할 닉네임
- `type`: `flyer`, `sms`, `receipt`, `priceTag`, `manual`
- `approvalStatus`: `pending`, `approved`, `rejected`, `needsRevision`, `duplicate`
- `martName`: 제보 대상 마트명
- `title`: 제보 제목 또는 대표 상품명
- `description`: 행사 기간, 가격, 수량 한정 등 추가 설명
- `imageUrls`: Storage 연결 후 사진 URL 저장
- `inputItems`: 상품명/가격/용량 등 관리자가 참고할 구조화 후보
- `adminMemo`: 승인/반려 메모
- `rewardPoint`: 즉시 지급 포인트가 아니라 필요 시 기여 점수/예정 포인트로 사용

### dealItems

홈 상품 카드에 노출되는 승인 특가 상품이다. 원본 `reports`와 연결되어야 한다.

주요 필드:

- `sourceReportId`: 원본 제보 id
- `sourceType`: 전단지/문자/사진/영수증
- `martName`
- `productName`
- `price`
- `saleStartDate`
- `saleEndDate`
- `badges`: `관리자 승인`, `전단지 확인`, `매장 사진 확인`, `영수증 확인`, `오늘 확인`, `수량 한정`
- `visible`: 홈 노출 여부

### flyers

동네마트 페이지에 노출되는 승인 전단지 원본이다.

주요 필드:

- `sourceReportId`
- `martName`
- `imageUrls`
- `saleStartDate`
- `saleEndDate`
- `approvedAt`
- `visible`

## 승인 정책

- 사진 없는 제보는 기본적으로 검토중 또는 수정 요청으로 둔다.
- 전단지/문자 제보는 1건만으로 자동 노출하지 않는다.
- 전단지/문자 유사 제보가 3건 이상이면 관리자 승인 추천 상태로 올린다.
- 매장 상품+가격표 사진은 관리자 확인 후 단건 승인 가능하다.
- 영수증은 구매 확인 배지에 활용하되, 현재 행사 지속 여부는 별도로 확인한다.

## 포인트 정책

- 승인 1건마다 즉시 현금성 포인트를 지급하지 않는다.
- 승인 제보는 앱 노출과 제보자 닉네임 표시로 기본 보상을 제공한다.
- 좋아요/담기/조회수는 주간 우수 제보 선정 참고 지표로만 사용한다.
- 포인트는 주간/월간 우수 제보자와 신규 제보자 추첨 등 예산 고정형 보상으로 운영한다.
