# 마트 할인정보 비교 서비스 기획서

## 1. 프로젝트 개요

이 서비스는 모바일 앱과 웹으로 구성된 마트 할인정보 비교 서비스이다. 사용자는 주변 마트의 할인 상품과 가격 정보를 검색하고, 같은 상품의 마트별 가격을 비교하여 더 저렴하게 장을 볼 수 있다.

초기 서비스 지역은 강원도 원주시로 시작하며, 이후 시/군/구 단위 확장을 거쳐 전국 서비스로 확대한다.

초기 정보 수집은 전단지, 할인 문자, 영수증, 가격표 사진, 사용자 제보를 중심으로 진행한다. 이후 관리자 직접 입력, 마트 담당자 입력, 마트 전산 시스템 연동, 온라인몰 가격 비교까지 확장한다.

## 2. 서비스 목표

### 2.1 핵심 목표

- 원주시 지역 식자재마트와 동네 대형마트의 할인 정보를 한곳에 모은다.
- 사용자가 상품명을 검색하면 마트별 가격과 할인 정보를 비교해서 보여준다.
- 승인된 제보 정보를 기반으로 신뢰도 있는 가격 데이터를 구축한다.
- 할인 재료와 보유 재료를 바탕으로 저렴한 메뉴와 레시피를 추천한다.
- 장보기 메모와 할인 알림을 통해 사용자의 장보기 비용을 줄인다.

### 2.2 대상 마트

1차 대상:
- 원주시 식자재마트
- 원주시 동네 대형마트

추후 확장 대상:
- 이마트, 홈플러스, 롯데마트 등 브랜드 마트
- 쿠팡 로켓프레시
- 기타 온라인 신선식품몰
- 마트 전산 시스템 연동 가능 제휴 마트

## 3. 기능별 기획

### 3.1 상품 검색 기반 가격 비교 기능

목적:
- 사용자가 상품명을 검색하면 주변 마트별 가격/할인 정보를 비교해 보여준다.
- 최저가 상품과 구매 가능한 마트를 빠르게 찾도록 돕는다.

기본 흐름:
1. 사용자가 상품명을 검색한다.
2. 검색 결과에서 관련 상품 목록을 확인한다.
3. 상품을 선택하면 마트별 가격 정보가 표시된다.
4. 사용자는 가격, 거리, 할인율, 행사 기간을 기준으로 비교한다.
5. 원하는 마트를 선택하면 해당 마트의 할인 정보 페이지로 이동한다.

표시 정보:
- 상품명
- 브랜드
- 용량/단위
- 판매 가격
- 할인 가격
- 할인율
- 행사 기간
- 판매 마트
- 마트와의 거리
- 최저가 여부
- 정보 출처

정렬 기준:
- 최저가순
- 가까운 거리순
- 할인율 높은순
- 최신 등록순

MVP 범위:
- 상품명 검색
- 마트별 가격 비교
- 최저가 표시
- 마트 상세 할인 페이지 연결

### 3.2 마트별 할인정보 기능

목적:
- 사용자가 특정 마트를 선택하면 해당 마트의 현재 할인 상품을 확인할 수 있다.

주요 기능:
- 마트별 할인 상품 목록
- 카테고리별 필터
- 행사 기간 표시
- 전단지 이미지 보기
- 마트 위치/길찾기
- 주차 가능 여부
- 영업시간
- 전화번호
- 즐겨찾기 마트 등록

MVP 범위:
- 마트 기본 정보
- 할인 상품 목록
- 전단지 이미지
- 위치/길찾기

### 3.3 오늘의 특가/광고 영역

목적:
- 지역 전체에서 할인율이 큰 상품 또는 광고 상품을 노출한다.
- 이후 광고 상품, 제휴 마트 상품, 추천 상품 노출 영역으로 확장한다.

초기 역할:
- 검색/마트 화면 상단 추천 영역
- 원주시 기준 할인율 높은 상품 표시
- 인기 검색 상품 표시

추후 확장:
- 광고 상품 등록
- 제휴 마트 우선 노출
- 지역별 특가 배너
- AI 추천 특가 상품

MVP 범위:
- 할인율 높은 상품 자동 노출
- 관리자 추천 상품 수동 등록

### 3.4 제보/포인트 기능

목적:
- 사용자가 전단지, 문자, 영수증, 가격표 사진을 제보하면 관리자가 승인하고 포인트를 지급한다.

제보 종류:
- 전단지 사진
- 할인 문자 캡처
- 영수증 사진
- 상품 가격표 사진
- 상품 정보 직접 입력

제보 처리 원칙:
- 제보된 정보는 관리자 승인 전에는 사용자에게 노출하지 않는다.
- 승인 완료된 정보만 가격 비교와 마트별 할인정보에 노출한다.
- 중복 정보는 포인트 미지급 또는 일부 지급 처리할 수 있다.
- 허위 정보는 반려 및 사용자 제한 대상으로 관리한다.

포인트 지급 기준:
- 승인된 가격 정보에 대해 포인트를 지급한다.
- 승인된 영수증, 전단지, 할인 문자, 가격표 제보에 포인트를 지급한다.
- 유용한 레시피나 생활 정보가 채택되면 추가 포인트를 지급할 수 있다.

포인트 교환:
- 초기에는 내부 포인트로 운영한다.
- 일정 포인트 이상부터 네이버페이 포인트 등 외부 포인트 교환 신청이 가능하도록 설계한다.
- 1차 MVP에서는 교환 신청과 관리자 수동 처리 방식으로 시작할 수 있다.

MVP 범위:
- 제보 등록
- 관리자 승인/반려
- 승인 시 포인트 적립
- 포인트 내역 확인

### 3.5 레시피 추천 기능

목적:
- 사용자의 보유 재료와 주변 마트 할인 재료를 바탕으로 저렴한 요리/메뉴를 추천한다.

추천 방식:
- 사용자가 가진 재료 기반
- 오늘 할인 중인 재료 기반
- 예산 기반
- 냉장고 재료 + 할인 재료 조합
- AI 추천
- 사전 등록 레시피 DB 기반 추천

주요 기능:
- 보유 재료 저장
- 기본 재료 저장
- 예산 입력
- 인분 선택
- 조리 난이도 선택
- 추천 레시피 확인
- 부족한 재료 표시
- 부족한 재료의 최저가 마트 표시
- 예상 총 장보기 비용 계산

기본 재료 예시:
- 간장
- 소금
- 설탕
- 식용유
- 고춧가루
- 된장
- 고추장

추천 예시:
- 1만원 이하 저녁
- 오늘 할인 중인 돼지고기로 만들 수 있는 요리
- 집에 양파, 계란이 있을 때 가능한 요리
- 냉장고 재료 + 원주시 할인 재료 조합 추천

MVP 범위:
- 보유 재료 입력
- 할인 재료 기반 추천
- 예산 기반 추천

### 3.6 장보기 메모/할인 알림 기능

목적:
- 사용자가 사야 할 상품을 메모해두면, 해당 상품이 할인될 때 알림을 받는다.

주요 기능:
- 장보기 메모 작성
- 상품명 입력
- 수량 입력
- 구매 완료 체크
- 메모 상품과 할인 상품 자동 매칭
- 할인 알림
- 즐겨찾기 마트 기준 알림 설정
- 희망 가격 이하 알림 설정

알림 조건:
- 메모에 등록한 상품이 승인된 할인 정보에 올라온 경우
- 사용자가 설정한 희망 가격 이하로 내려간 경우
- 사용자의 기본 지역 또는 즐겨찾기 마트에 해당 상품이 등록된 경우

MVP 범위:
- 장보기 메모 작성
- 구매 완료 체크
- 상품명 기반 할인 매칭
- 푸시 알림 준비

### 3.7 영수증 기반 자동 가계부 기능

목적:
- 사용자가 영수증을 등록하면 식비 소비 기록을 자동으로 정리한다.

- 마트별 지출 확인
- 카테고리별 지출 확인
- 지난달 대비 증감 확인
- 자주 구매한 상품 분석

표시 정보:
- 구매일
- 마트명
- 총 결제 금액
- 상품명
- 상품별 가격
- 카테고리
- 결제 수단

MVP 범위:
- 영수증 업로드
- 총 금액/마트명/날짜 저장
- 일별/월별 지출 통계
- 수동 수정

## 5. Firebase 컬렉션 구조 기획

### 5.1 기본 설계 방향

사용 기술:
- Flutter 모바일 앱
- Firebase Authentication
- Cloud Firestore
- Firebase Storage
- Cloud Functions
- Firebase Cloud Messaging

기본 원칙:
- 사용자 인증은 Firebase Authentication을 사용한다.
- 앱 주요 데이터는 Cloud Firestore에 저장한다.
- 전단지, 영수증, 가격표 사진은 Firebase Storage에 저장한다.
- 알림, 포인트 지급, OCR 처리, 통계 집계는 Cloud Functions로 확장한다.
- 사용자에게 노출되는 가격 정보는 관리자 승인 완료 데이터만 사용한다.

### 5.2 주요 컬렉션

users:
- 사용자 계정 정보
- 로그인 제공자, 닉네임, 기본 지역, 포인트, 등급 관리

marts:
- 마트 정보
- 마트명, 주소, 위치, 영업시간, 주차 여부, 휴무일 관리

products:
- 공통 상품 DB
- 상품명, 카테고리, 대표 단위, 검색 키워드 관리

price_items:
- 마트별 가격/할인 정보
- 특정 상품이 어느 마트에서 얼마에 판매되는지 저장

reports:
- 사용자 제보 정보
- 전단지, 문자, 영수증, 가격표 사진 제보 관리

points:
- 포인트 적립/차감 내역
- 승인 제보, 교환 신청, 운영자 조정 기록

shopping_memos:
- 장보기 메모
- 사용자가 등록한 구매 예정 상품과 알림 조건 관리

receipts:
- 영수증 소비 기록
- OCR 결과, 구매일, 총액, 상품별 소비 기록 관리

recipes:
- 레시피 정보
- 재료, 예산, 난이도, 추천 조건 관리

notifications:
- 알림 기록
- 할인 알림, 제보 승인 알림, 포인트 알림 관리

### 5.3 공통 상품 DB + 마트별 가격 정보 분리

상품 정보는 products에 한 번 등록하고, 마트별 가격 정보는 price_items에 따로 저장한다.

예시:

```text
products
- productId: product_green_onion
- name: 대파
- category: 채소
- keywords: 대파, 파
- defaultUnit: 단

price_items
- productId: product_green_onion
- martId: mart_001
- martName: 원주OO식자재마트
- salePrice: 1500
- originalPrice: 2500
- unit: 단
- saleStartDate: 2026-04-28
- saleEndDate: 2026-05-01
- source: 전단지
- approvalStatus: 승인 완료
```

이 구조의 장점:
- 같은 상품의 마트별 가격 비교가 쉽다.
- 상품명 검색과 가격 데이터 관리가 분리된다.
- 이후 쿠팡 로켓프레시, 대형마트, 마트 전산 연동을 붙이기 쉽다.

### 5.4 상품 검색 정책

초기 검색:
- 상품명 기준 단순 검색
- products.name
- products.keywords
- price_items.productName

추후 확장:
- 유사어 검색
- 오타 보정
- 초성 검색
- 카테고리 필터
- 인기 검색어
- 최근 검색어

유사어 예시:
- 대파 = 파
- 계란 = 달걀
- 돼지고기 앞다리살 = 앞다리 = 전지
- 삼겹살 = 돼지삼겹
- 양배추 = 통양배추

운영 방식:
- 초기에는 관리자가 직접 keywords를 등록한다.
- 검색 데이터가 쌓이면 자주 검색되는 단어를 기준으로 유사어를 보강한다.

### 5.5 마트 담당자 입력 기능 정책

1차 MVP:
- 서비스 관리자만 상품/가격/할인 정보를 직접 등록할 수 있다.
- 사용자 제보는 관리자 승인 후 노출한다.
- 마트 담당자 계정은 1차 출시에서는 제외한다.

추후 확장:
- 마트 담당자 계정 생성
- 담당 마트만 관리 가능
- 할인 상품 직접 등록
- 전단지 직접 업로드
- 행사 기간 관리
- 관리자 최종 승인 또는 자동 승인 정책 선택

   - 상품 메모 작성
   - 구매 완료 체크
   - 할인 상품과 매칭
   - 할인 알림 준비
